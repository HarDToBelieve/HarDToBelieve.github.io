30/*!
31@protocol
32@abstract JSExport provides a declarative way to export Objective-C objects and
33 classes -- including properties, instance methods, class methods, and
34 initializers -- to JavaScript.
35
36@discussion When an Objective-C object is exported to JavaScript, a JavaScript
37 wrapper object is created.
38
39 In JavaScript, inheritance works via a chain of prototype objects.
40 For each Objective-C class in each JSContext, an object appropriate for use
41 as a prototype will be provided. For the class NSObject the prototype
42 will be the Object prototype. For all other Objective-C
43 classes a prototype will be created. The prototype for a given
44 Objective-C class will have its internal [Prototype] property set to point to
45 the prototype created for the Objective-C class's superclass. As such the
46 prototype chain for a JavaScript wrapper object will reflect the wrapped
47 Objective-C type's inheritance hierarchy.
48
49 JavaScriptCore also produces a constructor for each Objective-C class. The
50 constructor has a property named 'prototype' that references the prototype,
51 and the prototype has a property named 'constructor' that references the
52 constructor.
53
54 By default JavaScriptCore does not export any methods or properties from an
55 Objective-C class to JavaScript; however methods and properties may be exported
56 explicitly using JSExport. For each protocol that a class conforms to, if the
57 protocol incorporates the protocol JSExport, JavaScriptCore exports the methods
58 and properties in that protocol to JavaScript
59
60 For each exported instance method JavaScriptCore will assign a corresponding
61 JavaScript function to the prototype. For each exported Objective-C property
62 JavaScriptCore will assign a corresponding JavaScript accessor to the prototype.
63 For each exported class method JavaScriptCore will assign a corresponding
64 JavaScript function to the constructor. For example:
65
66<pre>
67@textblock
68 @protocol MyClassJavaScriptMethods <JSExport>
69 - (void)foo;
70 @end
71
72 @interface MyClass : NSObject <MyClassJavaScriptMethods>
73 - (void)foo;
74 - (void)bar;
75 @end
76@/textblock
77</pre>
78
79 Data properties that are created on the prototype or constructor objects have
80 the attributes: <code>writable:true</code>, <code>enumerable:false</code>, <code>configurable:true</code>.
81 Accessor properties have the attributes: <code>enumerable:false</code> and <code>configurable:true</code>.
82
83 If an instance of <code>MyClass</code> is converted to a JavaScript value, the resulting
84 wrapper object will (via its prototype) export the method <code>foo</code> to JavaScript,
85 since the class conforms to the <code>MyClassJavaScriptMethods</code> protocol, and this
86 protocol incorporates <code>JSExport</code>. <code>bar</code> will not be exported.
87
88 JSExport supports properties, arguments, and return values of the following types:
89
90 Primitive numbers: signed values up to 32-bits convert using JSValue's
91 valueWithInt32/toInt32. Unsigned values up to 32-bits convert using JSValue's
92 valueWithUInt32/toUInt32. All other numeric values convert using JSValue's
93 valueWithDouble/toDouble.
94
95 BOOL: values convert using JSValue's valueWithBool/toBool.
96
97 id: values convert using JSValue's valueWithObject/toObject.
98
99 Objective-C instance pointers: Pointers convert using JSValue's
100 valueWithObjectOfClass/toObject.
101
102 C structs: C structs for CGPoint, NSRange, CGRect, and CGSize convert using
103 JSValue's appropriate methods. Other C structs are not supported.
104
105 Blocks: Blocks convert using JSValue's valueWithObject/toObject.
106
107 All objects that conform to JSExport convert to JavaScript wrapper objects,
108 even if they subclass classes that would otherwise behave differently. For
109 example, if a subclass of NSString conforms to JSExport, it converts to
110 JavaScript as a wrapper object rather than a JavaScript string.
111*/
112@protocol JSExport
113@end

115/*!
116@define
117@abstract Rename a selector when it's exported to JavaScript.
118@discussion When a selector that takes one or more arguments is converted to a JavaScript
119 property name, by default a property name will be generated by performing the
120 following conversion:
121
122 - All colons are removed from the selector
123
124 - Any lowercase letter that had followed a colon will be capitalized.
125
126 Under the default conversion a selector <code>doFoo:withBar:</code> will be exported as
127 <code>doFooWithBar</code>. The default conversion may be overridden using the JSExportAs
128 macro, for example to export a method <code>doFoo:withBar:</code> as <code>doFoo</code>:
129
130<pre>
131@textblock
132 @protocol MyClassJavaScriptMethods <JSExport>
133 JSExportAs(doFoo,
134 - (void)doFoo:(id)foo withBar:(id)bar
135 );
136 @end
137@/textblock
138</pre>
139
140 Note that the JSExport macro may only be applied to a selector that takes one
141 or more argument.
142*/
143#define JSExportAs(PropertyName, Selector) \
144 @optional Selector __JS_EXPORT_AS__##PropertyName:(id)argument; @required Selector
