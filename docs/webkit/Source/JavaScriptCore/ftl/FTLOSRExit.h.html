70struct OSRExitDescriptor {
71private:
72 WTF_MAKE_NONCOPYABLE(OSRExitDescriptor);
73public:
74 OSRExitDescriptor(
75 DataFormat profileDataFormat, MethodOfGettingAValueProfile,
76 unsigned numberOfArguments, unsigned numberOfLocals, unsigned numberOfTmps);
77
78 // The first argument to the exit call may be a value we wish to profile.
79 // If that's the case, the format will be not Invalid and we'll have a
80 // method of getting a value profile. Note that all of the ExitArgument's
81 // are already aware of this possible off-by-one, so there is no need to
82 // correct them.
83 DataFormat m_profileDataFormat;
84 MethodOfGettingAValueProfile m_valueProfile;
85
86 FixedOperands<ExitValue> m_values;
87 Bag<ExitTimeObjectMaterialization> m_materializations;
88
89 void validateReferences(const TrackedReferences&);
90
91 // Call this once we have a place to emit the OSR exit jump and we have data about how the state
92 // should be recovered. This effectively emits code that does the exit, though the code is really a
93 // patchable jump and we emit the real code lazily. The description of how to emit the real code is
94 // up to the OSRExit object, which this creates. Note that it's OK to drop the OSRExitHandle object
95 // on the ground. It contains information that is mostly not useful if you use this API, since after
96 // this call, the OSRExit is simply ready to go.
97 Ref<OSRExitHandle> emitOSRExit(
98 State&, ExitKind, const DFG::NodeOrigin&, CCallHelpers&, const B3::StackmapGenerationParams&,
99 uint32_t dfgNodeIndex, unsigned offset);
100
101 // In some cases you want an OSRExit to come into existence, but you don't want to emit it right now.
102 // This will emit the OSR exit in a late path. You can't be sure exactly when that will happen, but
103 // you know that it will be done by the time late path emission is done. So, a linker task will
104 // surely happen after that. You can use the OSRExitHandle to retrieve the exit's label.
105 //
106 // This API is meant to be used for things like exception handling, where some patchpoint wants to
107 // have a place to jump to for OSR exit. It doesn't care where that OSR exit is emitted so long as it
108 // eventually gets access to its label.
109 Ref<OSRExitHandle> emitOSRExitLater(
110 State&, ExitKind, const DFG::NodeOrigin&, const B3::StackmapGenerationParams&,
111 uint32_t dfgNodeIndex, unsigned offset);
112
113private:
114 // This is the low-level interface. It will create a handle representing the desire to emit code for
115 // an OSR exit. You can call OSRExitHandle::emitExitThunk() once you have a place to emit it. Note
116 // that the above two APIs are written in terms of this and OSRExitHandle::emitExitThunk().
117 Ref<OSRExitHandle> prepareOSRExitHandle(
118 State&, ExitKind, const DFG::NodeOrigin&, const B3::StackmapGenerationParams&,
119 uint32_t dfgNodeIndex, unsigned offset);
120};

135 }
