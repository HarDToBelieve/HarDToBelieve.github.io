120 // Each cell has a built-in lock. Currently it's simply available for use if you need it. It's
121 // a full-blown WTF::Lock. Note that this lock is currently used in JSArray and that lock's
122 // ordering with the Structure lock is that the Structure lock must be acquired first.
123
124 // We use this abstraction to make it easier to grep for places where we lock cells.
125 // to lock a cell you can just do:
126 // Locker locker { cell->cellLocker() };
127 JSCellLock& cellLock() { return *reinterpret_cast<JSCellLock*>(this); }

150 // Returns information about how to call/construct this cell as a function/constructor. May tell
151 // you that the cell is not callable or constructor (default is that it's not either). If it
152 // says that the function is callable, and the OverridesGetCallData type flag is set, and
153 // this is an object, then typeof will return "function" instead of "object". These methods
154 // cannot change their minds and must be thread-safe. They are sometimes called from compiler
155 // threads.

200 bool atomicCompareExchangeCellStateWeakRelaxed(CellState oldState, CellState newState)
201 {
202 return WTF::atomicCompareExchangeWeakRelaxed(&m_cellState, oldState, newState);
203 }
204
205 CellState atomicCompareExchangeCellStateStrong(CellState oldState, CellState newState)
206 {
207 return WTF::atomicCompareExchangeStrong(&m_cellState, oldState, newState);
208 }
