123 // In some rare cases, we'll reuse a DOMWindow for a new Document. For example,
124 // when a script calls window.open("..."), the browser gives JavaScript a window
125 // synchronously but kicks off the load in the window asynchronously. Web sites
126 // expect that modifications that they make to the window object synchronously
127 // won't be blown away when the network load commits. To make that happen, we
128 // "securely transition" the existing DOMWindow to the Document that results from
129 // the network load. See also SecurityContext::isSecureTransitionTo.
130 void didSecureTransitionTo(Document&);
