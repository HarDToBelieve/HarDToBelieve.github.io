430 // FIXME: This method is retained because of embedded WebViews in AppKit. When a WebView is embedded inside
431 // some enclosing view with auto-pagination, no call happens to resize the view. The new pagination model
432 // needs the view to resize as a result of the breaks, but that means that the enclosing view has to potentially
433 // resize around that view. Auto-pagination uses the bounds of the actual view that's being printed to determine
434 // the edges of the print operation, so the resize is necessary if the enclosing view's bounds depend on the
435 // web document's bounds.
436 //
437 // This is already a problem if the view needs to be a different size because of printer fonts or because of print stylesheets.
438 // Mail/Dictionary work around this problem by using the _layoutForPrinting SPI
439 // to at least get print stylesheets and printer fonts into play, but since WebKit doesn't know about the page offset or
440 // page size, it can't actually paginate correctly during _layoutForPrinting.
441 //
442 // We can eventually move Mail to a newer SPI that would let them opt in to the layout-time pagination model,
443 // but that doesn't solve the general problem of how other AppKit views could opt in to the better model.
444 //
445 // NO OTHER PLATFORM BESIDES MAC SHOULD USE THIS METHOD.

452 // Coordinate systems:
453 //
454 // "View"
455 // Top left is top left of the FrameView/ScrollView/Widget. Size is Widget::boundsRect().size().
456 //
457 // "TotalContents"
458 // Relative to ScrollView's scrolled contents, including headers and footers. Size is totalContentsSize().
459 //
460 // "Contents"
461 // Relative to ScrollView's scrolled contents, excluding headers and footers, so top left is top left of the scroll view's
462 // document, and size is contentsSize().
463 //
464 // "Absolute"
465 // Relative to the document's scroll origin (non-zero for RTL documents), but affected by page zoom and page scale. Mostly used
466 // in rendering code.
467 //
468 // "Document"
469 // Relative to the document's scroll origin, but not affected by page zoom or page scale. Size is equivalent to CSS pixel dimensions.
470 // FIXME: some uses are affected by page zoom (e.g. layout and visual viewports).
471 //
472 // "Client"
473 // Relative to the visible part of the document (or, more strictly, the layout viewport rect), and with the same scaling
474 // as Document coordinates, i.e. matching CSS pixels. Affected by scroll origin.
475 //
476 // "LayoutViewport"
477 // Similar to client coordinates, but affected by page zoom (but not page scale).
478 //

575 // Page and FrameView both store a Pagination value. Page::pagination() is set only by API,
576 // and FrameView::pagination() is set only by CSS. Page::pagination() will affect all

987inline void FrameView::incrementVisuallyNonEmptyPixelCount(const IntSize& size)
988{
989 if (m_visuallyNonEmptyPixelCount > visualPixelThreshold)
990 return;
991
992 auto area = size.area<RecordOverflow>() + m_visuallyNonEmptyPixelCount;
993 if (UNLIKELY(area.hasOverflowed()))
994 m_visuallyNonEmptyPixelCount = std::numeric_limits<decltype(m_visuallyNonEmptyPixelCount)>::max();
995 else
996 m_visuallyNonEmptyPixelCount = area;
997}
