30namespace JSC {
31
32enum CompilationResult {
33 // We tried to compile the code, but we couldn't compile it. This could be
34 // because we ran out of memory, or because the compiler encountered an
35 // internal error and decided to bail out gracefully. Either way, this implies
36 // that we shouldn't try to compile this code block again.
37 CompilationFailed,
38
39 // The profiling assumptions that were fed into the compiler were invalidated
40 // even before we finished compiling. This means we should try again: in such
41 // cases the profiling will now be updated and the next compilation won't
42 // encounter the same problem. But it does mean that we should exercise
43 // exponential back-off, to get even more profiling so that new profiling
44 // pathologies aren't encountered.
45 CompilationInvalidated,
46
47 // The compilation succeeded and the code block now has JITCode for the newly
48 // compiled code. However, compilation success doesn't mean that the CodeBlock
49 // will execute yet; you typically have to install it first, unless you plan
50 // on invoking it manually (something that *could* be done for some kinds of
51 // OSR entry).
52 CompilationSuccessful,
53
54 // We decided to do the compilation asynchronously. This means that we haven't
55 // yet compiled the code. This only happens when you pass a
56 // DeferredCompilationCallback. That callback will get called with some
57 // interesting result, once compilation completes.
58 CompilationDeferred
59};
60
61} // namespace JSC
