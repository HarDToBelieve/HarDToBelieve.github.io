30namespace JSC { namespace B3 { namespace Air {
31
32enum class StackSlotKind : uint8_t {
33 // A locked stack slot is an area of stack requested by the client. It cannot be killed. The
34 // client can get its FP offset and write to it from stack walking code, so we must assume
35 // that reads and writes to a locked stack slot can be clobbered the same way as reads and
36 // writes to any memory location.
37 Locked,
38
39 // A spill slot. These have fundamentally different behavior than a typical memory location.
40 // They are lowered to from temporaries. This means for example that a 32-bit ZDef store to a
41 // 8 byte stack slot will zero the top 4 bytes, even though a 32-bit ZDef store to any other
42 // kind of memory location would do no such thing. UseAddr on a spill slot is not allowed, so
43 // they never escape.
44 Spill
45
46 // FIXME: We should add a third mode, which means that the stack slot will be read asynchronously
47 // as with Locked, but never written to asynchronously. Then, Air could optimize spilling and
48 // filling by tracking whether the value had been stored to a read-only locked slot. If it had,
49 // then we can refill from that slot.
50 // https://bugs.webkit.org/show_bug.cgi?id=150587
51};
52
53} } } // namespace JSC::B3::Air
