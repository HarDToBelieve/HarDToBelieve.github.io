48// How do JSC VM options work?
49// ===========================
50// The FOR_EACH_JSC_OPTION() macro below defines a list of all JSC options in use,
51// along with their types and default values. The options values are actually
52// realized as fields in OptionsStorage embedded in JSC::Config.
53//
54// Options::initialize() will initialize the option values with the defaults
55// specified in FOR_EACH_JSC_OPTION() below. After that, the values can be
56// programmatically read and written to using an accessor method with the same
57// name as the option. For example, the option "useJIT" can be read and set like
58// so:
59//
60// bool jitIsOn = Options::useJIT(); // Get the option value.
61// Options::useJIT() = false; // Sets the option value.
62//
63// If you want to tweak any of these values programmatically for testing
64// purposes, you can do so in Options::initialize() after the default values
65// are set.
66//
67// Alternatively, you can override the default values by specifying
68// environment variables of the form: JSC_<name of JSC option>.
69//
70// Note: Options::initialize() tries to ensure some sanity on the option values
71// which are set by doing some range checks, and value corrections. These
72// checks are done after the option values are set. If you alter the option
73// values after the sanity checks (for your own testing), then you're liable to
74// ensure that the new values set are sane and reasonable for your own run.
75//
76// Any modifications to options must be done before the first VM is instantiated.
77// On instantiation of the first VM instance, the Options will be write protected
78// and cannot be modified thereafter.
