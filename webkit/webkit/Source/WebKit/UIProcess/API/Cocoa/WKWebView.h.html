237 @discussion The completionHandler is passed the result of the script evaluation or an error.
238 Calling this method is equivalent to calling `evaluateJavaScript:inFrame:inContentWorld:completionHandler:` with:
239 - A `frame` value of `nil` to represent the main frame
240 - A `contentWorld` value of `WKContentWorld.pageWorld`
241*/
242- (void)evaluateJavaScript:(NSString *)javaScriptString completionHandler:(void (^ _Nullable)(_Nullable id, NSError * _Nullable error))completionHandler;

244/* @abstract Evaluates the given JavaScript string.
245 @param javaScriptString The JavaScript string to evaluate.
246 @param frame A WKFrameInfo identifying the frame in which to evaluate the JavaScript string.
247 @param contentWorld The WKContentWorld in which to evaluate the JavaScript string.
248 @param completionHandler A block to invoke when script evaluation completes or fails.
249 @discussion The completionHandler is passed the result of the script evaluation or an error.
250
251 Passing nil is equivalent to targeting the main frame.
252 If the frame argument no longer represents a valid frame by the time WebKit attempts to call the JavaScript function your completion handler will be called with a WKErrorJavaScriptInvalidFrameTarget error.
253 This might happen for a number of reasons, including but not limited to:
254 - The target frame has been removed from the DOM via JavaScript
255 - A parent frame has navigated, destroying all of its previous child frames
256
257 No matter which WKContentWorld you use to evaluate your JavaScript string, you can make changes to the underlying web content. (e.g. the Document and its DOM structure)
258 Such changes will be visible to script executing in all WKContentWorlds.
259 Evaluating your JavaScript string can leave behind other changes to global state visibile to JavaScript. (e.g. `window.myVariable = 1;`)
260 Those changes will only be visibile to scripts executed in the same WKContentWorld.
261 evaluateJavaScript: is a great way to set up global state for future JavaScript execution in a given world. (e.g. Importing libraries/utilities that future JavaScript execution will rely on)
262 Once your global state is set up, consider using callAsyncJavaScript: for more flexible interaction with the JavaScript programming model.
263*/
264- (void)evaluateJavaScript:(NSString *)javaScriptString inFrame:(nullable WKFrameInfo *)frame inContentWorld:(WKContentWorld *)contentWorld completionHandler:(void (^ _Nullable)(_Nullable id, NSError * _Nullable error))completionHandler NS_REFINED_FOR_SWIFT WK_API_AVAILABLE(macos(11.0), ios(14.0));

266/* @abstract Calls the given JavaScript string as an async JavaScript function, passing the given named arguments to that function.
267 @param functionBody The JavaScript string to use as the function body.
268 @param arguments A dictionary representing the arguments to be passed to the function call.
269 @param frame A WKFrameInfo identifying the frame in which to call the JavaScript function.
270 @param contentWorld The WKContentWorld in which to call the JavaScript function.
271 @param completionHandler A block to invoke with the return value of the function call, or with the asynchronous resolution of the function's return value.
272 @discussion The functionBody string is treated as an anonymous JavaScript function body that can be called with named arguments.
273 Do not format your functionBody string as a function-like callable object as you would in pure JavaScript.
274 Your functionBody string should contain only the function body you want executed.
275
276 For example do not pass in the string:
277 @textblock
278 function(x, y, z)
279 {
280 return x ? y : z;
281 }
282 @/textblock
283
284 Instead pass in the string:
285 @textblock
286 return x ? y : z;
287 @/textblock
288
289 The arguments dictionary supplies the values for those arguments which are serialized into JavaScript equivalents.
290 For example:
291 @textblock
292 @{ @"x" : @YES, @"y" : @1, @"z" : @"hello world" };
293 @/textblock
294
295 Combining the above arguments dictionary with the above functionBody string, a function with the arguments named "x", "y", and "z" is called with values true, 1, and "hello world" respectively.
296
297 Allowed argument types are:
298 NSNumber, NSString, NSDate, NSArray, NSDictionary, and NSNull.
299 Any NSArray or NSDictionary containers can only contain objects of those types.
300
301 Passing nil is equivalent to targeting the main frame.
302 If the frame argument no longer represents a valid frame by the time WebKit attempts to call the JavaScript function your completion handler will be called with a WKErrorJavaScriptInvalidFrameTarget error.
303 This might happen for a number of reasons, including but not limited to:
304 - The target frame has been removed from the DOM via JavaScript
305 - A parent frame has navigated, destroying all of its previous child frames
306
307 No matter which WKContentWorld you use to call your JavaScript function, you can make changes to the underlying web content. (e.g. the Document and its DOM structure)
308 Such changes will be visible to script executing in all WKContentWorlds.
309 Calling your JavaScript function can leave behind other changes to global state visibile to JavaScript. (e.g. `window.myVariable = 1;`)
310 Those changes will only be visibile to scripts executed in the same WKContentWorld.
311
312 Your completion handler will be called with the explicit return value of your JavaScript function.
313 If your JavaScript does not explicitly return any value, that undefined result manifests as nil being passed to your completion handler.
314 If your JavaScript returns null, that result manifests as NSNull being passed to your completion handler.
315
316 JavaScript has the concept of a "thenable" object, which is any JavaScript object that has a callable "then" property.
317 The most well known example of a "thenable" object is a JavaScript promise.
318 If your JavaScript returns a "thenable" object WebKit will call "then" on the resulting object and wait for it to be resolved.
319
320 If the object resolves successfully (e.g. Calls the "fulfill" function) your completion handler will be called with the result.
321 If the object rejects (e.g. Calls the "reject" function) your completion handler will be called with a WKErrorJavaScriptAsyncFunctionResultRejected error containing the reject reason in the userInfo dictionary.
322 If the object is garbage collected before it is resolved, your completion handler will be called with a WKErrorJavaScriptAsyncFunctionResultUnreachable error indicating that it will never be resolved.
323
324 Since the function is a JavaScript "async" function you can use JavaScript "await" on thenable objects inside your function body.
325 For example:
326 @textblock
327 var p = new Promise(function (f) {
328 window.setTimeout("f(42)", 1000);
329 });
330 await p;
331 return p;
332 @/textblock
333
334 The above function text will create a promise that will fulfull with the value 42 after a one second delay, wait for it to resolve, then return the fulfillment value of 42.
335*/
336#ifdef NS_SWIFT_ASYNC_NAME
337- (void)callAsyncJavaScript:(NSString *)functionBody arguments:(nullable NSDictionary<NSString *, id> *)arguments inFrame:(nullable WKFrameInfo *)frame inContentWorld:(WKContentWorld *)contentWorld completionHandler:(void (^ _Nullable)(_Nullable_result id, NSError * _Nullable error))completionHandler NS_REFINED_FOR_SWIFT WK_API_AVAILABLE(macos(11.0), ios(14.0));
338#else
339- (void)callAsyncJavaScript:(NSString *)functionBody arguments:(nullable NSDictionary<NSString *, id> *)arguments inFrame:(nullable WKFrameInfo *)frame inContentWorld:(WKContentWorld *)contentWorld completionHandler:(void (^ _Nullable)(_Nullable id, NSError * _Nullable error))completionHandler NS_REFINED_FOR_SWIFT WK_API_AVAILABLE(macos(11.0), ios(14.0));
340#endif
