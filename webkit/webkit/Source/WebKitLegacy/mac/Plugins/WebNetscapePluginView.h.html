119// -willCallPlugInFunction must be called before calling any of the NPP_* functions for this view's plugin.
120// This is necessary to ensure that plug-ins are not destroyed while WebKit calls into them. Some plug-ins (Flash
121// at least) are written with the assumption that nothing they do in their plug-in functions can cause NPP_Destroy()
122// to be called. Unfortunately, this is not true, especially if the plug-in uses NPN_Invoke() to execute a
123// document.write(), which clears the document and destroys the plug-in.
124// See <rdar://problem/4480737>.
125- (void)willCallPlugInFunction;
126
127// -didCallPlugInFunction should be called after returning from a plug-in function. It should be called exactly
128// once for every call to -willCallPlugInFunction.
129// See <rdar://problem/4480737>.
