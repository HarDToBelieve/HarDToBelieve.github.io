32/*!
33@enum WebCacheModel
34
35@abstract Specifies a usage model for a WebView, which WebKit will use to
36determine its caching behavior.
37
38@constant WebCacheModelDocumentViewer Appropriate for a WebView displaying
39a fixed document -- like a splash screen, a chat document, or a word processing
40document -- with no UI for navigation. The WebView will behave like any other
41view, releasing resources when they are no longer referenced. Remote resources,
42if any, will be cached to disk. This is the most memory-efficient setting.
43
44Examples: iChat, Mail, TextMate, Growl.
45
46@constant WebCacheModelDocumentBrowser Appropriate for a WebView displaying
47a browsable series of documents with a UI for navigating between them -- for
48example, a reference materials browser or a website designer. The WebView will
49cache a reasonable number of resources and previously viewed documents in
50memory and/or on disk.
51
52Examples: Dictionary, Help Viewer, Coda.
53
54@constant WebCacheModelPrimaryWebBrowser Appropriate for a WebView in the
55application that acts as the user's primary web browser. The WebView will cache
56a very large number of resources and previously viewed documents in memory
57and/or on disk.
58
59Examples: Safari, OmniWeb, Shiira.
60*/
61typedef NS_ENUM(NSUInteger, WebCacheModel) {
62 WebCacheModelDocumentViewer = 0,
63 WebCacheModelDocumentBrowser = 1,
64 WebCacheModelPrimaryWebBrowser = 2
65} WEBKIT_ENUM_DEPRECATED_MAC(10_5, 10_14);

245/*!
246 @property cacheModel
247 @abstract Specifies a usage model for a WebView, which WebKit will use to
248 determine its caching behavior. If necessary, WebKit
249 will prune its caches to match cacheModel when set.
250
251 @discussion Research indicates that users tend to browse within clusters of
252 documents that hold resources in common, and to revisit previously visited
253 documents. WebKit and the frameworks below it include built-in caches that take
254 advantage of these patterns, substantially improving document load speed in
255 browsing situations. The WebKit cache model controls the behaviors of all of
256 these caches, including NSURLCache and the various WebCore caches.
257
258 Applications with a browsing interface can improve document load speed
259 substantially by specifying WebCacheModelDocumentBrowser. Applications without
260 a browsing interface can reduce memory usage substantially by specifying
261 WebCacheModelDocumentViewer.
262
263 If cacheModel is not set, WebKit will select a cache model automatically.
264*/
265@property (nonatomic) WebCacheModel cacheModel;
