72// The Widget class serves as a base class for three kinds of objects:
73// (1) Scrollable areas (ScrollView)
74// (2) Scrollbars (Scrollbar)
75// (3) Plugins (PluginView)
76//
77// A widget may or may not be backed by a platform-specific object (e.g., HWND on Windows, NSView on Mac, QWidget on Qt).
78//
79// Widgets are connected in a hierarchy, with the restriction that plugins and scrollbars are always leaves of the
80// tree. Only ScrollViews can have children (and therefore the Widget class has no concept of children).
81//
82// The rules right now for which widgets get platform-specific objects are as follows:
83// ScrollView - Mac
84// Scrollbar - Mac, Gtk
85// Plugin - Mac, Windows (windowed only), Qt (windowed only, widget is an HWND on windows), Gtk (windowed only)
86//
